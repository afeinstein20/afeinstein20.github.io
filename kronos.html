A Guide to Reading and Using our Data Products


We will upload all data products as High Level Science Products (HLSP) on the Mikulski Archive for Space Telescopes (MAST). We opt to put our data on MAST instead of Zenodo because it allows users to download individual files, instead of Zenodo where users must download the entire directory. All of our data products are stored as h5py files. Below, we provide a description of each data product, the file naming convention, and what key words are within the file. The units, while listed below, are also stored in the header of each data product file, under the same keyword.


Stellar spectra files - We provide the extracted stellar spectra as a function of time for all of our independent reductions. The file name is presented as follows:
kronos-[author's last name]-[pipeline]-spectra.h5
Within each file are the following key words:

  • time: Time, BJD.
  • wavelength: Wavelength solution, micron.
  • stellar_flux: Stellar spectra, counts/s.
  • stellar_flux_err: Stellar spectra errors, counts/s.


    • Light curve files - We provide the extracted white light and spectroscopic light curves for all of our independent reductions, along with all of the best-fit models. The file name is presented as follows:
    kronos-[author's last name]-[pipeline]-lcs.h5
    Within each file are the following key words:

  • time: Time, BJD.
  • wavelength: Wavelengths corresponding to the light curves, micron.
  • flux_wlc: White light curve, normalized flux.
  • flux_err_wlc: White light curve error, normalized flux.
  • model_wlc: Best-fit model to the white light curve, normalized flux.
  • flux_{0}: Spectroscopic light curve at a given wavelength, normalized flux.
  • flux_err_{0}: Spectroscopic light curve errors at a given wavelength, normalized flux.
  • model_{0}: Best-fit model to the spectroscopic light curve at a given wavelength, normalized flux.
